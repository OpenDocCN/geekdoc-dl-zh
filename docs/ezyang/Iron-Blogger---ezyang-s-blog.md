# 铁博客：ezyang 的博客

> 来源：[`blog.ezyang.com/2009/12/iron-blogger/`](http://blog.ezyang.com/2009/12/iron-blogger/)

这个社会实验已经在[Google 的第四位](http://www.google.com/search?q=iron+blogger)搜索结果中名列前茅，“铁博客”的排名还会更高。[铁博客](http://web.mit.edu/nelhage/www/iron-blogger.html)是一个关于啤酒（对我来说并非如此）、博客和同行压力的实验。

既然我们谈论博客及其为何人们（包括我自己）似乎无法做到的原因，我们不妨回顾一下我多年来曾经尝试但未能成功的博客。

我做的第一个博客是在 Blogger 还是一个闪亮的新服务、博客刚刚在 2004 年成为主流的时候。这个博客被命名为[低语图书馆](http://blog.thewritingpot.com/)，最初的版本非常像是一个疯子的胡言乱语。糟糕的双关语层出不穷。那是“写作锅”的时代。我想在这段时间里渴望得到反馈和交流，但是当半规律的发布时间未能引起任何评论时，这个博客逐渐荒废了。

2007 年，我将博客重新用作[HTML 净化器](http://htmlpurifier.org/)的开发日志，希望 HTML 净化器的众多用户能从中获得开发过程的一些见解。*那*是建筑宇航员的繁荣时代，是在 PHP 中编写 Java 和设计模式大行其道的时期。我还偶尔写些糟糕的诗。同样，缺乏反馈和回应使得博客逐渐荒废。此外，我已经学会非常厌恶 Blogger 的界面及其对换行 HTML 的无能。

在 2009 年夏天，我为自己定下了一个宏伟的目标，每周写一篇实质性的博客文章；每篇文章都将被精心撰写和修订，内容为王，作为一名麻省理工学院的学生，我肯定有些话是世界关心的。开篇文章将如下所述：

> 写作即表达。
> 
> 它是将你脑海中边缘的东西结合到字母、单词、句子、段落、章节的结构中去。它是锁定你想说的意义，但同时打开你还可以说但尚未说出口的可能性。
> 
> 因此，请不要将这看作一个网络日志，因为如果它是一个日志，船长会定期回到分类账并重写历史的涂鸦以使其变得漂亮。把它看作是一本文章集，每篇文章都是我想说的东西的概述。

但计划被写作难度极大的事实所搁置，事实上，博客并不是一个很合适的媒介（如果你在写一篇阐述性文章，你很希望能保持更新），而且 WordPress 的代码，坦率地说，相当令人毛骨悚然。

Iron Blogger，不仅仅是由于无聊而带来的“威胁”经济负担，还因为它具有观察功能：除非至少有人模糊地关注你的博客，否则它是不能发挥作用的。因此，我很自豪地开始了这个宏大的实验。让无聊开始吧！
